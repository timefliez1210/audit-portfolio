# [000041] Invalid validation allows creation of 11 pools instead of 10
  
  ### Summary

The `createPool()` function is intended to enforce a maximum of 10 pools per bidding project, but the check is incorrectly implemented:
```solidity
require(biddingProject.poolCount <= 10, Cannot_Exceed_Ten_Pools_Per_Project());
```

Because the condition uses `<=` instead of `<`, a project with `poolCount == 10` is still allowed to create one more pool, resulting in 11 total pools, which violates the protocol's documented limit.

### Root Cause

The check in the `require(biddingProject.poolCount <= 10, Cannot_Exceed_Ten_Pools_Per_Project());`  but it must be `require(biddingProject.poolCount < 10, Cannot_Exceed_Ten_Pools_Per_Project());`, https://github.com/dualguard/2025-11-alignerz/blob/main/protocol/src/contracts/vesting/AlignerzVesting.sol#L689.
