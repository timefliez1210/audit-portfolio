# [000871] Underflow on `claimDividends()` when `startTime` is in the future
  
  ### Summary

`startTime` may be freely set in the future. If that's the case, a call to `claimDividends()` will revert with an underflow, instead of a clean error message:
```solidity
            secondsPassed = block.timestamp - startTime;
```
